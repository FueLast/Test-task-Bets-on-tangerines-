# 🟠 MandarinBid

> Real-time auction platform built with ASP.NET Core, demonstrating background processing, concurrency control, and clean service-oriented architecture.

---

## 📌 Overview

**MandarinBid** — это веб-приложение аукциона, где пользователи в реальном времени делают ставки на лоты (мандаринки) с ограниченным временем жизни.

Проект спроектирован с акцентом на:
- корректную обработку конкурентных запросов (ставки)
- разделение ответственности (Controller → Service → Data)
- асинхронную обработку фоновых задач
- устойчивость к ошибкам (retry, logging, validation)

---

## ⚙️ Tech Stack

- **ASP.NET Core MVC**
- **Entity Framework Core**
- **ASP.NET Identity**
- **SQL Server**
- **Background Services (IHostedService)**
- **Mailtrap API (email delivery)**
- **Dependency Injection (built-in)**

---

## 🧠 Architecture

Проект реализован в классическом layered-стиле:

### 🔹 Controllers
- принимают HTTP-запросы
- выполняют базовую валидацию
- передают управление сервисам

### 🔹 Services
- содержат бизнес-логику приложения
- работают с БД через EF Core
- реализуют сценарии аукциона (ставки, уведомления)

### 🔹 Background Services
- выполняют асинхронные задачи вне HTTP-потока
- обеспечивают масштабируемость и отзывчивость системы

### 🔹 Data Layer
- `ApplicationDbContext`
- модели: `Mandarin`, `Bid`
- конфигурация precision и типов данных

## 🔥 Key Features

- 📈 **Real-time bidding**
  - обновление интерфейса каждые 5 секунд

- ⚔️ **Concurrency-safe ставки**
  - защита от race conditions
  - optimistic concurrency (`RowVersion`)

- 🧠 **Бизнес-валидация**
  - запрет самоперебивания
  - проверка минимальной ставки
  - блокировка ставок после окончания

- 📬 **Асинхронные уведомления**
  - уведомление при перебитой ставке
  - уведомление победителя

- 🔁 **Retry механизм**
  - повторные попытки отправки email

- ⚙️ **Background processing**
  - генерация лотов
  - завершение аукционов
  - обработка очереди задач

---

## ⚠️ Concurrency Handling

Обработка конкурентных ставок реализована через комбинацию:

- `RowVersion` (optimistic concurrency)
- проверку актуальной цены
- обработку `DbUpdateConcurrencyException`

Это гарантирует, что:
- пользователь не может перебить уже обновлённую ставку
- данные остаются консистентными при высокой нагрузке

## 🔄 Background Processing

### 📌 Используемые сервисы:

- `MandarinGeneratorService`
  - автоматически создаёт новые лоты

- `MandarinCleanupService`
  - завершает аукционы
  - определяет победителя
  - инициирует отправку email

- `EmailBackgroundService`
  - обрабатывает очередь задач
  - выполняет отправку писем вне HTTP-запроса

---

## 📩 Email Flow

1. пользователь делает ставку  
2. предыдущий лидер определяется  
3. задача отправки email добавляется в очередь  
4. background service обрабатывает задачу  
5. email отправляется через Mailtrap API  

---

## 🧩 Design Decisions & Trade-offs

- **Queue вместо прямой отправки email**
  - улучшает производительность
  - исключает блокировку HTTP-запроса

- **AsNoTracking для чтения**
  - снижает нагрузку на EF Core

- **UserName сохраняется в Bid**
  - уменьшает количество join-запросов
  - ускоряет отображение UI

- **Short-lived auctions**
  - упрощает тестирование и демонстрацию системы

---

## 🧪 Edge Cases Covered

- попытка сделать ставку после завершения аукциона  
- попытка перебить свою же ставку  
- ставка ниже текущей цены  
- конкурентные обновления (race condition)  
- отсутствие email у пользователя  
- сбои при отправке email
